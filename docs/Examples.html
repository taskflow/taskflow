This page collects self-contained examples that demonstrate Taskflow across a broad range of parallel programming patterns — from classic graph algorithms and numerical computing to dynamic task graphs, async pipelines, and GPU acceleration. Each example is designed to be read independently and includes a problem formulation, a parallel implementation, and a discussion of the key Taskflow features at work.

Task Graph

These examples demonstrate how to express parallelism as a static task dependency graph using tf::Taskflow, including conditional and non-deterministic control flow.

  • Wavefront Parallelism — diagonal sweep parallelism on a 2D grid
  • Nondeterministic Control Flow — stochastic control flow with conditional tasks
  • Graph Traversal — parallel DAG traversal driven by graph structure

Parallel Algorithms

These examples apply Taskflow's parallel algorithms and pipeline scheduling to classic computational and data-processing problems.

  • Matrix Multiplication — parallel matrix multiplication on CPU
  • k-means Clustering — parallel k-means clustering on CPU
  • Text Processing Pipeline — character frequency analysis as a three-stage pipeline
  • Graph Processing Pipeline — wavefront pipeline over a DAG
  • Taskflow Processing Pipeline — nested taskflow graphs inside pipeline stages

GPU Task Graph

These examples demonstrate how to offload computation to a CUDA GPU using tf::cudaGraph, coordinated with CPU tasks in a single taskflow.

  • Matrix Multiplication with CUDA GPU — GPU-accelerated matrix multiplication
  • k-means Clustering with CUDA GPU — GPU-accelerated k-means clustering

Async Tasking

These examples focus on Taskflow's dynamic task creation capabilities — async tasks, dependent-async task graphs, cooperative execution, and recursive runtime tasking.

  • Fibonacci Number — recursive parallelism with runtime tasking and cooperative synchronization
  • Async Producer-Consumer Pipeline — overlapping producer, validator, and consumer stages with dependent-async tasks
  • Divide-and-Conquer Parallelism — divide-and-conquer parallelism with tf::TaskGroup, using merge sort as a concrete example
  • Dynamic Dependency Graph — runtime-driven task graph whose topology depends on program data
